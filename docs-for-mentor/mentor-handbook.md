# メンターハンドブック

> **本書は devquest‑suite のメンター向けに、哲学・ワークフロー・実務ツールをまとめたガイドです。**  

---

## 1. ミッションと対象範囲
- **ミッション**: 素晴らしいソフトウェアエンジニアを育てる
- **対象**: devquest‑suite のメンティ（ジュニア〜ミッドのソフトウェアエンジニア）。  
- **メンターの役割**: ガイド・コードレビュー・心理的安全性の確保・進捗管理。

---

## 2. コア原則
1. **成長マインドセット（Stretch 30 % ルール）**  
   – タスクは現在の能力で 70 % 成功、30 % 挑戦となる難度に設定。  
2. **問いかけ型コーチング**  
   – *「どのようなトレードオフを検討しましたか？」* などのオープンクエスチョンを活用。  
3. **SBI＋改善提案フィードバック**  
   – *Situation → Behavior → Impact → Suggestion* で具体的に伝える。

---

## 3. メンタリングワークフロー
```
1‑on‑1 → タスク選定 → 実装 → PR & CI → レビュー → レトロ
```
| フェーズ | メンターのチェックリスト |
|----------|---------------------------|
| 1‑on‑1 | progress‑tracker を確認しブロッカー抽出・週次目標設定 |
| タスク選定 | 難度タグ・期限を合意 |
| PR & CI | CI がグリーンになるのを確認後レビュー開始 |
| レビュー | コードレビュー チェックリストを使用 |
| レトロ | 成功点・改善点を振り返り次のアクションを設定 |

---

## 4. コミュニケーションガイド
- **言葉遣い**: 敬意・具体性・"I ステートメント" を心掛ける  
- **テンプレート**: `session-agenda-template.md`, `feedback-phrases.md` を参照

---

## 5. タスク割当て戦略
- 2 連続でタスク達成したら難度を上げる

---

## 6. 評価とルーブリック
- 詳細は [`evaluation-rubric.md`](evaluation-rubric.md) を参照。  
- **合格ライン**: 重み付きスコア ≧ 80 % またはメンター裁量。  

---

## 7. 1‑on‑1 アジェンダテンプレート
```md
### 1‑on‑1 — <date>
- **今週の成果 (Wins)**:
- **課題 / ブロッカー**:
- **次週目標**:
- **メンターへの要望**:
```

---

## 8. コードレビュー チェックリスト
- [ ] 設計がタスク README と整合している  
- [ ] 正確性（全テストパス）  
- [ ] 命名・可読性  
- [ ] 複雑度が仕様内  
- [ ] Docstring・コメント十分  
- [ ] エッジケースを含むテスト (カバレッジ ≥ 80 %)

---

## 9. 生成 AI の安全な利用
- **禁止データ**: 機密コード・シークレット・個人情報  

---

## 10. 進捗トラッキング
- 各タスク完了後 `progress-tracker.md` を更新  
- CI が自動付与するテスト結果を週次確認

---

## 11. 継続的改善
- 四半期ごとにメンターレトロを実施  
- ハンドブック更新は `docs-change` ラベル付き PR で提出
