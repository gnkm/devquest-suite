## メンタリングプラン（16 週間ロードマップ）

| フェーズ | 何を学ぶか | なぜ学ぶか | どう学ぶか | 目安時間* |
|---------|------------|-----------|------------|-----------|
| **0. キックオフ (Week 0)** | ✅ SMART 目標設定<br>✅ devquest‑suite 環境セットアップ | 学習の指標と土台を明確にする | 1‑on‑1 で目標合意 → `mentee-profile.toml` 作成 | 2 h |
| **1. コーディング基礎 & TDD (Weeks 1‑4)** | - Python イディオム<br>- アルゴリズム初級（探索・ソート）<br>- テスト駆動開発 | バグを早期に防ぎ、再現性あるコードを書く力を養う | *Tasks*: `01-algo`, `02-python`<br>*Method*: TDD → PR → レビュー | 6 h/週 |
| **2. ソフトウェア設計 & クリーンコード (Weeks 5‑6)** | - SOLID / クリーンアーキテクチャ<br>- リファクタリング手法 | 可読性・保守性を高め、変更コストを下げる | 設計課題 (`03-design`) + 既存コードのリファクタリングペアプロ | 5 h/週 |
| **3. DevOps & 自動化 (Weeks 7‑9)** | - Docker / GitHub Actions CI<br>- モニタリング基礎 | “動くソフト”を継続的に届けるパイプラインを構築 | *Tasks*: `04-devops`<br>CI で自動テスト→Slack 通知 | 6 h/週 |
| **4. 生成 AI 活用入門 (Weeks 10‑11)** | - プロンプトエンジニアリング基礎<br>- GPT, Claude, Gemini, Copilot でのコード生成 | AI で実装速度を上げ、設計に集中するため | 実装課題を AI で草案 → 手動リファクタ → 評価 (`05-ai-basics`) | 5 h/週 |
| **5. AI × コード品質 (Weeks 12‑13)** | - AI 生成コードのレビュー指針<br>- セキュリティ・ライセンスリスク | AI 出力を安全・高品質に取り込む判断力を持つ | ケーススタディ + `code-review-checklist.md` でレビュー演習 | 4 h/週 |
| **6. 応用プロジェクト (Weeks 14‑15)** | - 小規模 API サービスをゼロから実装<br>- AI によるリファクタ提案の統合 | 習得スキルをワンストップで実践 & ポートフォリオ化 | チーム形式または個人開発、メンターがアーキレビュー | 8 h/週 |
| **7. 総括 & 次期目標設定 (Week 16)** | - レーダーチャートによるスキル再評価<br>- 生成 AI 上級ロードマップ策定 | 成長を可視化し、次の挑戦領域を決定 | 1‑on‑1 Retro → 新 SMART 目標を Tommy | 2 h |

\*時間は自己学習＋実装＋1‑on‑1 を含む目安（平日 1 h + 週末 4‑5 h ペース）

---

### 学習方法の詳細

1. **devquest‑suite Tasks**  
   - 自動テスト付きの課題を解いてプルリク提出 → CI → メンターが `code-review-checklist.md` でレビュー  
   - 生成 AI の利用は *why / prompt / 修正内容* を PR に記録し、振り返りで議論

2. **Weekly 1‑on‑1**  
   - `session-agenda-template.md` を事前に下書きし、Wins・Challenges を共有  
   - フィードバックは **SBI + 改善提案** フォーマット

3. **ペアプロ & Simulated Pair‑AI**  
   - 難易度の高い箇所は VS Code Live Share でペアプロ  
   - 同じ手順を Copilot Chat とも実施し、思考プロセスを比較 → メタ認知を促進

4. **マイクロ講義（15 分）**  
   - 要点のみの講義 → すぐに関連タスクへ着手  
   - 集中 → 即応用のリズムで定着率向上

5. **Progress Tracker**  
   - `progress-tracker.csv` に自動採点結果 + メンター評価を毎週追加  
   - レーダーチャートを CI が生成 → 可視化でモチベ維持

### ゴール到達時のアウトカム

| 技術軸 | 成果物・指標 |
|-------|--------------|
| **コード品質** | コードレビュー指摘 1 回平均 < 3 点、テストカバレッジ ≥ 80 % |
| **DevOps** | Docker 化＋CI/CD パイプラインをゼロから構築 |
| **生成 AI** | AI 生成コードをレビューし、再利用できる比率 ≥ 50 % |
| **ソフトスキル** | 1‑on‑1 での自己報告 & フィードバック文書化が定着 |

16 週間終了時点で **「自力で設計し、CI で品質を担保しつつ、AI を安全に加速装置として使えるエンジニア」** になることを目指します。
