# 対話型生成 AI を用いた学習を行うための運用フロー

## 1. ドキュメントの3レイヤー構造

| レイヤー | 目的 | フォーマット例 | 更新頻度 |
| --- | --- | --- | --- |
| **(1) プロフィール (Static)** | 自己紹介・長期目標・現在のスキルセット | `mentee-profile.toml` | 1 〜 3 か月に1回 |
| **(2) ロードマップ & バックログ (Iterative)** | 学習テーマ・完了済/未完了タスク・期限 | `learning-roadmap.md` | 週1回程度 |
| **(3) セッション履歴 (Session)** | *原文*と*圧縮サマリ*のペアを保存 | `sessions/2025-04-20.raw.md`  `sessions/2025-04-20.summary.md` | 毎セッション |

> **圧縮サマリ**は「次回 AI に渡す部分」。長くても **250 token / 1 KB 程度**に抑え、後述のテンプレでそのまま貼れる粒度にしておきます。

---

## 2. 1 回のセッション・フロー

1. **Kick‑off**  
   - テンプレートに(1)(2)と最新の(3)(summary)を貼り付けて新セッションを開始。
   - AI には「この情報だけを前提とし、追加で聞き返して可読性を損なわないようにしてほしい」と明示。

2. **対話・実践**  
   - コードレビュー・設計相談・課題実装などを行う。
   - 必要に応じて(2)ロードマップをリアルタイムで更新。

3. **クローズ**  
   - AI に以下を依頼し、(3)summary を生成して保存。  
     - 学習トピックと得た知見（5 行以内）  
     - 直近 TODO（3 項目以内、粒度は 30 〜 120 分）  
     - ブロッカーや疑問点（あれば）  
     - 文字数／トークン上限
   - 追加で(1)と(2)を更新する場合は同時に反映。

4. **(オプ) 自動化**  
   - GitHub Actions / Makefile で  
     `make next-session` →  
     `cat mentee-profile.toml roadmap.md last.summary.md > prompt.txt`  
     のように生成するだけでも十分省力化できます。

---

## 3. 新セッション用プロンプト・テンプレート

```text
### あなたの役割
あなたは私の生成AIメンターです。プロフェッショナルかつ建設的なフィードバックを重視してください。

### 学習者プロフィール (抜粋: mentee-profile.toml)
name      = "Genki Matsunaga"
primary_goal = "生成AIを活用できる一流ソフトウェアエンジニアになる"
experience = { python = "4 年", typescript = "1 年" }
interests  = ["LLM プロンプト設計", "クリーンアーキテクチャ", "DevOps"]

### 現在のロードマップ (learning-roadmap.md 要約)
- 4 月: テスト駆動開発 (完了 60%)
- 5 月: クリーンアーキテクチャ入門
- 6 月: 本番運用のための Observability 基礎

### 前回セッションサマリ (2025‑04‑18)
1. テストダブルの種類 (Stub / Mock / Fake) を整理  
2. GitHub Actions で pytest を並列化 → 実行時間 40% 削減  
3. 次回までの TODO  
   - [ ] サンプル API に Mock を導入してみる (90 min)  
   - [ ] pytest‑xdist の flaky test 対策を調査 (60 min)

### 今日お願いしたいこと
- 上記 TODO の振り返り
- クリーンアーキテクチャの“エンティティ層”実装例を TypeScript で示してほしい
- 不明点があれば質問してください
```

> **使い方**: (1)(2)(3) を貼り替えて冒頭に貼るだけ。どの AI でも同一のコンテキストが共有できます。

---

## 4. 継続学習を加速させる小ワザ

| 工夫 | 期待効果 |
| --- | --- |
| **トークン監視**: LLM に「300 token を超えそうなら要約し直す」と依頼 | “サマリが膨れ上がる”問題を防止 |
| **タグ運用**: session ファイル名に `TDD`, `LLM` などタグを含める | 検索と並列学習テーマ管理 |
| **定期レビュー**: 月末に(1)(2)(3) を読み返し、古い TODO や不要情報を削除 | プロンプト肥大化を抑制 |
| **自動リンク生成**: `summary.md` に GitHub/Notion の該当コードや記事 URL を埋め込む | 復習コスト削減 |
| **マルチ AI 比較学習**: 同じ prompt を別 AI に投げて回答を diff | 発想の幅を広げ、ベストプラクティスを抽出 |

---

### まとめ

- **Static→Iterative→Session** の3層で情報を整理すると「毎回渡すべき最小限」が明確になる  
- 各セッションの**圧縮サマリ**が“次のプロンプトの種”  
- テンプレートを固定すれば、モデルやブラウザを変えても学習が途切れない  
- スクリプト化すればほぼワンクリック運用

このフローを回しながら、学習テーマごとにロードマップを更新していけば、生成 AI をメンターにした“継続学習パイプライン”が完成します。試行してみて運用コストや粒度を微調整してみてください。
